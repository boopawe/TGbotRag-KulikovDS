## RAG Bot для Telegram. 

Telegram‑бот, реализующий метод Retrieval‑Augmented Generation (RAG) для ответов на вопросы пользователей. Бот использует лёгкую языковую модель **TinyLlama‑1.1B‑Chat** и векторную базу данных **ChromaDB** для хранения и семантического поиска фактов.

### Почему TinyLlama?

- Лёгкая модель (1.1B параметров) — не требует мощных вычислений.  
- Быстрая генерация ответов (примерно 10–30 секунд на CPU).  
- Полностью локальная работа — не требует интернета после установки.  
- Отличное соотношение скорости и качества для RAG‑задач.

### Назначение данной LLM модели:
- TinyLlama-1.1B-Chat — это компактная и эффективная диалоговая версия языковой модели TinyLlama, созданная для работы на устройствах с ограниченными вычислительными ресурсами (смартфоны, ноутбуки, Raspberry Pi)
- Архитектура: упрощённая версия **Llama 2**
- Базовая модель была обучена на 3 триллионах токенов. Чат-версия дообучена на диалоговых датасетах (например, UltraChat, OpenAssistant) с использованием методик, похожих на Zephyr от Hugging Face

## Основные возможности

- **Локальная LLM**: генерация ответов с использованием модели TinyLlama‑1.1B‑Chat.  
- **Векторная база знаний**: хранение фактов и семантический поиск с помощью ChromaDB.  
- **RAG‑система**: автоматический поиск релевантного контекста в базе перед генерацией ответа.  
- **Управление данными**: команды для добавления, просмотра и очистки базы знаний.  
  
## Содержимое базы знаний (астрофизика)

По умолчанию база содержит следующие факты:

1. Масса сверхмассивной чёрной дыры в центре Млечного Пути (Стрелец A*) составляет около 4 миллионов масс Солнца.
2. Самая быстро вращающаяся нейтронная звезда (пульсар PSR J1748-2446ad) делает 716 оборотов в секунду.
3. Расстояние до ближайшей к Солнцу звезды (Проксима Центавра) — 4,24 световых года.
4. Возраст Вселенной, согласно последним данным спутника Planck, составляет 13,8 миллиардов лет.
5. На Юпитере регулярно выпадают алмазные дожди.
6. Гамма-всплески — самые мощные взрывы во Вселенной, за секунду они выделяют столько же энергии, сколько Солнце за всю свою жизнь.

Список фактов можно расширять через команду `/add`.

## 3. Листинги кода (ключевые фрагменты)

### `database.py` — векторная база знаний (ChromaDB)

- **Инициализация клиента и коллекции**  
  ```python
  self.client = chromadb.PersistentClient(path=storage_path)
  self.collection = self.client.get_collection(self.collection_name)
  ```
  Хранит векторную БД на диске и работает с конкретной collection, где есть факты.

- **Добавление нового факта**
  ```python
  record_id = str(uuid.uuid4())
  self.collection.add(documents=[text_content], ids=[record_id])
  ```
  Каждый фрагмент текста сохраняется как документ с уникальным `id`. 

- **Поиск похожих фактов**
  ```python
  search_results = self.collection.query(query_texts=[query_text], n_results=results_count)
  return "\n".join(search_results['documents'])
  ```
  По запросу пользователя ищет ближайшие по векторам фрагменты и возвращает их как контекст для LLM.

- **Получение всех записей и очистка базы**
  ```python
  all_data = self.collection.get()
  formatted_records = [f"{idx}. {record}" for idx, record in enumerate(all_data['documents'], 1)]
  return "\n".join(formatted_records)

  all_ids = self.collection.get()['ids']
  self.collection.delete(ids=all_ids)
  ```
  Позволяют отдельной командой `/get_all` смотреть весь контент базы и `/clear_db` — полностью очистить её.


### `llm.py` — интерфейс к LLM‑модели TinyLlama‑1.1B

- **Загрузка модели и токенайзера**
  ```python
  self.token_processor = AutoTokenizer.from_pretrained(model_path)
  self.language_model = AutoModelForCausalLM.from_pretrained(
      model_path,
      torch_dtype=torch.float16 if torch.cuda.is_available() else torch.float32
  )
  self.language_model = self.language_model.to(self.device)
  ```
  Загружает локальную модель `TinyLlama/TinyLlama-1.1B-Chat-v1.0` на CPU или GPU в half‑precision для экономии памяти.

- **Формирование промпта с системным инструктажем**
  ```python
  formatted_prompt = f"""<|system|>
  Ты полезный ассистент. Отвечай подробно и развёрнуто.</s>
  <|user|>
  {user_input}</s>
  <|assistant|>"""
  ```
  Использует чат‑формат токенов, который модель ожидает: `system` → `user` → `assistant`.

- **Генерация ответа с параметрами**
  ```python
  generated_outputs = self.language_model.generate(
      **input_tensors,
      max_new_tokens=max_length,
      temperature=0.7,
      do_sample=True,
      top_p=0.95,
      top_k=50,
      repetition_penalty=1.15
  )
  ```
  Параметры генерации.

- **Обрезка и фильтрация результата**
  ```python
  if len(final_response) > 500:
      final_response = final_response[:500] + "..."
  ```
  Ограничение ответа по символам для более стабильной работы.


### `run.py` — Основной модуль.

- **Команды бота (основа RAG‑интерфейса)**
  ```python
  commands_list = [
      BotCommand(command="add", description="Добавить факт: /add <текст>"),
      BotCommand(command="get_all", description="Показать все факты"),
      BotCommand(command="generate", description="Ответ ИИ без базы"),
      BotCommand(command="rag", description="Ответ с поиском"),
      BotCommand(command="clear_db", description="Очистить базу данных")
  ]
  ```
  Команды для взамодействия с ботом.

- **Добавление факта в базу**
  ```python
  @dp.message(Command("add"))
  async def add_fact(message: Message):
      fact = message.text.replace("/add", "").strip()
      knowledge_base.store_information(fact)
  ```
  Любой введённый `/add ...` сохраняется в векторную базу как новый документ.

- **Прямой ответ LLM без RAG**
  ```python
  @dp.message(Command("generate"))
  async def generate(message: Message):
      prompt = message.text.replace("/generate", "").strip()
      answer = await asyncio.wait_for(
          asyncio.get_event_loop().run_in_executor(None, ai_model.get_response, prompt),
          timeout=45.0
      )
  ```
  Чистая генерация без подключение базы данных.

- **Режим RAG (с поиском по базе)**
  ```python
  context = knowledge_base.find_similar(query, results_count=2)
  prompt = f"""На основе следующей информации из базы знаний ответь на вопрос пользователя...
  Информация из базы знаний:
  {context}
  Вопрос пользователя: {query}
  """
  answer = ai_model.get_response(prompt)
  ```
  Сначала ищет 2 ближайших факта, формирует контекстный промпт и получает ответ LLM, наполенный контекстом БД.

- **Очистка базы**
  ```python
  @dp.message(Command("clear_db"))
  async def clear_db(message: Message):
      knowledge_base.clear_all_records()
  ```
  Полностью сбрасывает все векторные записи через `collection.delete(ids=all_ids)`.

### **Сравнительная таблица**

| Аспект | LLM (базовая модель) | RAG (с базой знаний) |
|--------|----------------------|-----------------------|
| **Фактическая точность** | Почти нет фактов, много абстрактных рассуждений и галлюцинаций. | Устойчиво даёт правильное расстояние до Проксимы Центавра ≈4,24 световых года (≈40 трлн км). |
| **Релевантность** | Ответы часто уходят в философию, город, видимость и не отвечают напрямую на вопрос. | Стартует с прямого ответа на запрос, потом добавляет факты из базы. |
| **Структура ответа** | Обрывистые, несвязные фразы, теряется мысль. | Более структурировано, с попыткой расширить ответ деталями из базы знаний. |
| **Качество данных** | Галлюцинации и вымышленные конструкции вроде "до ближайшей звезды нет". | Часто вводит нерелевантные или неточные данные (например, о пульсаре PSR J1748‑2446ad и расстоянии 300 млн км — это не так, он лежит в другой системе). |
| **Полезность для пользователя** | Мало полезное: больше творческий поток, чем фактический ответ. | Значительно выше: пользователь сразу получает число и понимание, что ближайшая звезда — Проксима Центавра. |
